# Report προόδου και σημεία προς διόρθωση

Η υλοποίηση έχει προχωρήσει αρκετά σε σχέση με το αρχικό στάδιο. Πλέον υπάρχουν βασικά δομικά στοιχεία της πτυχιακής:

* cosine baseline retrieval,
* passage-based semantic graph construction,
* kNN graph,
* mutual kNN graph,
* threshold graph,
* cluster-based graph,
* Personalized PageRank,
* k-step neighborhood expansion,
* HITS / shortest-path ως επιπλέον graph-based heuristics,
* βασικό graph analysis.

Άρα η εργασία δεν είναι πλέον στο στάδιο απλής εξερεύνησης. Έχεις φτιάξει αρκετό από τον βασικό κορμό. Ωστόσο, πριν θεωρηθούν τα αποτελέσματα αξιόπιστα, πρέπει να διορθωθούν μερικά σημαντικά ζητήματα.

---

## 1. Θετικά σημεία προόδου

### 1.1 Υπάρχει cosine baseline

Υπάρχει πλέον καθαρή λογική:

```text
query embedding → cosine similarity με όλα τα passage embeddings → top-k ranking
```

Αυτό είναι σωστό και αποτελεί το βασικό reference point της εργασίας.

---

### 1.2 Υπάρχουν passage-based semantic graphs

Οι γράφοι πλέον βασίζονται σε passages/documents ως κόμβους και similarity-based edges. Αυτό είναι σωστό για την πτυχιακή.

Υπάρχουν υλοποιήσεις για:

```text
kNN graph
mutual kNN graph
threshold graph
cluster-based graph
```

---

### 1.3 Έχεις διορθώσει το edge weight στο kNN / mutual kNN

Στο kNN και mutual kNN χρησιμοποιείται πλέον:

```python
sim = 1.0 - distance
```

όταν το metric είναι cosine.

Αυτό είναι σωστό, γιατί στο semantic graph το edge weight πρέπει να εκφράζει similarity/closeness και όχι distance.

---

### 1.4 Υπάρχει Personalized PageRank

Έχει υλοποιηθεί function για Personalized PageRank και υπάρχει wrapper για evaluation. Αυτό είναι πολύ σημαντικό, γιατί το PPR είναι ένας από τους βασικούς graph-based retrieval αλγορίθμους της πτυχιακής.

Άρα δεν λείπει πλέον το PPR. Αυτό που μένει είναι να ελεγχθεί σωστά και να αξιολογηθεί συστηματικά.

---

### 1.5 Υπάρχει k-step neighborhood expansion

Έχεις επίσης ξεκινήσει k-step expansion. Αυτό είναι επίσης στον πυρήνα της πτυχιακής.

Ωστόσο, εδώ υπάρχει ακόμη σοβαρό πιθανό bug στο score mapping, το οποίο πρέπει να διορθωθεί πριν χρησιμοποιηθούν τα αποτελέσματα.

---

### 1.6 Υπάρχει graph analysis

Υπολογίζονται πλέον βασικά graph statistics όπως:

```text
nodes
edges
density
average degree
connected components
communities
PageRank top nodes
```

Αυτό είναι καλή βάση για την ανάλυση της δομής των γράφων.

# 2. Κρίσιμα θέματα που παραμένουν

## 2.1 Το nDCG formulation δεν είναι ακόμη καθαρό

Στο `evaluation.py`, η συνάρτηση `nDCGk_score` εξακολουθεί να χρησιμοποιεί:

```python
return ndcg_score(ideal, relevance, k=k)
```

Αυτό θέλει διόρθωση ή τουλάχιστον πολύ προσεκτικό έλεγχο.

Η `ndcg_score(y_true, y_score, k=k)` περιμένει:

```text
y_true = πραγματικά relevance labels
y_score = scores που έδωσε το σύστημα
```

Δεν χρειάζεται να δώσουμε εμείς το ideal ranking ως πρώτο argument, γιατί η sklearn το υπολογίζει εσωτερικά.

Για το δικό μας binary retrieval setup είναι πιο καθαρό να υλοποιηθεί χειροκίνητο `ndcg_at_k`, όπου:

```text
1 = relevant passage
0 = non-relevant passage
```

και να υπολογίζονται χειροκίνητα:

```text
DCG
IDCG
nDCG = DCG / IDCG
```
---

## 2.2 Το k-step neighborhood expansion έχει ακόμη σοβαρό πιθανό bug

Στο `k_step_neighborhood_expansion`, υπάρχει ακόμη η λογική:

```python
for i, node_id in enumerate(ids):
    graph_score = graph_scores[node_id]
    score = alpha * sims[i] + (1 - alpha) * graph_score
```

Το πρόβλημα είναι ότι:

```text
i = index στη λίστα των neighbor nodes
sims[i] = similarity του document στη θέση i στο πλήρες corpus
```

Άρα το `sims[i]` μπορεί να μην αντιστοιχεί στο `node_id`.

Αυτό σημαίνει ότι μπορεί να δίνεται cosine score άλλου passage σε λάθος node.

### Σωστή λογική

Πρέπει να φτιαχτεί mapping:

```python
sims, documents, _ = sim_scores_for_query(query_id)
doc_to_sim = {doc_id: sim for doc_id, sim in zip(documents, sims)}
```

και μετά:

```python
cos_score = doc_to_sim.get(node_id, 0.0)
score = alpha * cos_score + (1 - alpha) * graph_score
```
---

## 2.3 Το ίδιο bug υπάρχει και στο shortest_path

Στο `shortest_path`, υπάρχει αντίστοιχη λογική:

```python
for i, node_id in enumerate(nodes_in_path):
    graph_score = graph_scores[node_id]
    score = alpha * sims[i] + (1 - alpha) * graph_score
```

Και εδώ το `i` είναι index στο `nodes_in_path`, όχι index στο πλήρες corpus.

Άρα πρέπει να γίνει η ίδια διόρθωση με `doc_to_sim`.

---

## 2.4 Το threshold graph υπάρχει, αλλά το grid έχει ακόμη typo

Στο grid υπάρχει key:

```python
"threshold_distance "
```

με κενό στο τέλος.

Όμως η function περιμένει:

```python
threshold_params["threshold_distance"]
```

χωρίς κενό.

Άρα πρέπει να γίνει:

```python
threshold_grid = {
    "threshold_distance": [0.3, 0.4, 0.5, 0.6]
}
```

---

## 2.5 Το threshold graph δεν φαίνεται να έχει ενταχθεί πλήρως στο graph generation grid

Υπάρχει function για threshold graph, αλλά στο `make_graphs()` φαίνεται ότι δίνεται έμφαση κυρίως σε:

```text
kNN
mutual kNN
clustering graph
```

Πρέπει να προστεθεί καθαρό loop και για threshold graphs.

Παράδειγμα:

```python
for threshold_combo in grid_combinations(threshold_grid):
    threshold_params = merge(BASE_THRESHOLD, threshold_combo)
    name = make_name(f"threshold_{pre_tag}", threshold_combo)
    ex.threshold_graph(c, threshold_params, preprocess, name, True)
```

---

## 2.6 Το experimental grid είναι ακόμη μεγάλο

Το grid περιλαμβάνει πολλά:

```text
πολλά kNN values
πολλά distance metrics
πολλά preprocessing combinations
πολλά KMeans settings
πολλά clustering variants
```

Αυτό μπορεί να παράγει πάρα πολλούς γράφους, χωρίς να είναι εύκολη η ερμηνεία.

### Πρώτο controlled grid

Για αρχή πρέπει να τρέξει μικρό grid:

```python
preprocess_combinations = [
    {"scaler": None, "pca": None}
]

knn_grid = {
    "n_neighbors": [5, 10, 20],
    "metric": ["cosine"]
}

threshold_grid = {
    "threshold_distance": [0.3, 0.4, 0.5, 0.6]
}

kmeans_grid = {
    "n_clusters": [5, 10, 20],
    "init": ["k-means++"]
}
```

Μετά, αν δουλέψει σωστά αυτό, μπορεί να επεκταθεί σε PCA/scaling και άλλα distance metrics.
