## 1. Έλεγξε τα retrieval metrics με toy examples

### Τρέχουσα κατάσταση

Στο `evaluation.py` τα metrics έχουν βελτιωθεί σημαντικά. Το `Recall@K` πλέον διαιρεί με το πλήθος των relevant documents και το `RR_score` ψάχνει το πρώτο relevant μέσα στα predictions.

```python
def recallk_score(predictions, correct_results, k):
    correct_set = set(correct_results)
    if len(correct_set) == 0:
        return 0.0

    correct_guess = 0.0
    for i, prediction in enumerate(predictions):
        if i >= k:
            break
        if prediction in correct_results:
            correct_guess += 1

    return correct_guess/len(correct_results)
```

```python
def RR_score(predictions, correct_results):
    correct_set = set(correct_results)
    for indx, prediction in enumerate(predictions):
        if prediction in correct_set:
            return 1/(indx + 1)

    return 0
```

### Τι πρέπει να κάνεις

Δεν αρκεί να φαίνεται σωστός ο κώδικας. Πρέπει να γράψεις μικρά toy tests ώστε να επιβεβαιώσεις ότι τα metrics δίνουν τις αναμενόμενες τιμές.

Παράδειγμα:

```python
predictions = ["A", "B", "C", "D"]
correct = ["C", "X"]

# Recall@4 = 1/2 = 0.5
# RR = 1/3
```

Πρόσθεσε tests για:

- relevant στην πρώτη θέση,
- relevant στη μέση του ranking,
- κανένα relevant στο ranking,
- πολλά relevant documents.

## 2. Πρόσεξε το nDCG formulation

### Τρέχων κώδικας

```python
def nDCGk_score(predictions, correct_results, k):
    correct_set = set(correct_results)

    relevance = np.array([[1 if p in correct_set else 0 for p in predictions]])

    # sort relevance descending
    ideal = np.sort(relevance)[:, ::-1]

    return ndcg_score(ideal, relevance, k=k)
```

### Θέμα

Η λογική binary relevance είναι σωστή ως πρώτη εκδοχή:

```text
relevant = 1
non-relevant = 0
```
Στη συνάρτηση `nDCGk_score` πρέπει να ελεγχθεί προσεκτικά η χρήση της `ndcg_score` της `sklearn`, γιατί τα arguments που δέχεται έχουν συγκεκριμένη σημασία.

Η `ndcg_score(y_true, y_score, k=k)` περιμένει:

* στο `y_true`: τα πραγματικά relevance labels των documents,
* στο `y_score`: τα scores που έδωσε το σύστημα retrieval.

Άρα δεν πρέπει να δίνουμε εμείς το ideal ranking ως πρώτο argument. Η `sklearn` υπολογίζει εσωτερικά το ideal ranking με βάση τα relevance labels. Στο δικό μας retrieval setup τα relevant passages είναι ουσιαστικά binary labels:

```text
1 = relevant passage
0 = non-relevant passage
```

Επομένως, για να αποφύγουμε λάθη στη χρήση της `ndcg_score`, είναι πιο καθαρό να υλοποιηθεί δική μας συνάρτηση `ndcg_at_k`, η οποία θα παίρνει το ranked list των predicted passage ids και τα `correct_results`, θα φτιάχνει relevance vector με 1/0, και θα υπολογίζει χειροκίνητα το DCG, το ideal DCG και τελικά το nDCG@K.
```python
ndcg_score(y_true, y_score, k=k)
```

Εδώ χρησιμοποιείς:

```python
ndcg_score(ideal, relevance, k=k)
```

Αυτό χρειάζεται προσοχή. Για απλό binary ranking evaluation μπορείς είτε να κρατήσεις δική σου καθαρή υλοποίηση nDCG είτε να φτιάξεις `y_true` και `y_score` πιο καθαρά.

### Τι πρέπει να κάνεις

Γράψε toy examples και έλεγξε:

```text
αν το relevant είναι στη θέση 1 → nDCG πρέπει να είναι υψηλό / 1.0
αν το relevant είναι χαμηλά → nDCG πρέπει να πέφτει
αν δεν υπάρχει relevant → nDCG πρέπει να είναι 0
```

## 3. Το kNN / mutual kNN edge weight διορθώθηκε, αλλά κράτα το ως κανόνα

### Τρέχων κώδικας

Στο `graph_construction.py` πλέον υπάρχει σωστή μετατροπή από cosine distance σε similarity:

```python
if knn_params["metric"] == "cosine":
    sim = 1.0 - distances[i][j]
elif knn_params["metric"] in ("euclidean", "minkowski", "manhattan"):
    sim = np.exp(-distances[i][j])
G.add_edge(node_id, ids[nearest], weight=sim)
```

και αντίστοιχα στο mutual kNN:

```python
if knn_params["metric"] == "cosine":
    sim = 1.0 - d[id][1][i]
elif knn_params["metric"] in ("euclidean", "minkowski", "manhattan"):
    sim = np.exp(-d[id][1][i])
G.add_edge(ids[id], ids[neighbor], weight=sim)
```

### Τι πρέπει να θυμάσαι

Σε semantic graph:

```text
μεγάλο weight = μεγάλη σημασιολογική ομοιότητα
μικρό weight = ασθενής σημασιολογική σχέση
```

Άρα ποτέ δεν βάζουμε raw cosine distance ως weight αν ο αλγόριθμος το ερμηνεύει ως strength.

## 4. Σοβαρό πιθανό bug στο k-step neighborhood expansion

### Τρέχων κώδικας

Στο `retrieval.py`:

```python
def k_step_neighborhood_expansion(graph, important_nodes, query_id, k, hops, alpha, sims):
    neighbors = get_neighbors(graph, important_nodes, hops)
    ids = list(neighbors.keys())

    sims, documents, query_correct_results = sim_scores_for_query(query_id)
    final_scores = []

    graph_scores = personalised_pagerank(graph, important_nodes, sims, 0, ids)

    for i, node_id in enumerate(ids):
        graph_score = graph_scores[node_id]
        score = alpha * sims[i] + (1 - alpha) * graph_score
        final_scores.append((node_id, score))
```

### Γιατί είναι πρόβλημα

Το `i` εδώ είναι index μέσα στη λίστα `ids`, δηλαδή μέσα στους neighbors. Όμως το `sims[i]` είναι index μέσα στη λίστα όλων των documents από το corpus.

Άρα μπορεί να δίνεις cosine similarity άλλου document σε λάθος node.

### Τι πρέπει να κάνεις

Πρέπει να φτιάξεις mapping από document id σε similarity:

```python
sims, documents, _ = sim_scores_for_query(query_id)
doc_to_sim = {doc_id: sim for doc_id, sim in zip(documents, sims)}
```

και μετά:

```python
cos_score = doc_to_sim.get(node_id, 0.0)
score = alpha * cos_score + (1 - alpha) * graph_score
```

Αυτό είναι κρίσιμο. Αν δεν διορθωθεί, τα k-step/fusion αποτελέσματα δεν είναι αξιόπιστα.

## 6. Το PPR θέλει safety checks

### Τρέχων κώδικας

```python
def personalised_pagerank(graph, ids, similarities, k, nodes=None):
    node_to_idx = {node: idx for idx, node in enumerate(graph.nodes())}
    adjacency = csr_matrix(nx.to_scipy_sparse_array(graph, format='csr'))

    total = sum(similarities)
    weights = {
        node_to_idx[node_id]: similarities[i] / total
        for i, node_id in enumerate(ids)
        if node_id in node_to_idx
    }
```

### Θέματα που πρέπει να ελεγχθούν

1. Τι γίνεται αν `total == 0`;
2. Τι γίνεται αν κανένα seed node δεν υπάρχει στον graph;
3. Οι `similarities` που περνάς στη συνάρτηση αντιστοιχούν όντως στα `ids`;

### Τι πρέπει να κάνεις

Πρόσθεσε checks:

```python
valid = [(node_id, similarities[i]) for i, node_id in enumerate(ids) if node_id in node_to_idx]

total = sum(score for _, score in valid)
if total == 0 or len(valid) == 0:
    return []
```

και μετά φτιάξε το personalization vector μόνο από valid seed nodes.
